# frost
> Временные рабочие пространства с снапшотами. Весь мусор уходит сам — сохраняй только нужное
```
F        R        O  S         T
FROST is Recovery Of Stateless Trees
```

## Идея
`frost` — утилита что должна помочь с созданием временных директорий с tmpfs удобным и быстрым способом. Всё что делается внутри frost директорий существует только до перезагрузки и не засаряет лишним мусором вашу домашнюю директорию, всё сохраняется в удобные snapshot'ы: в ручную или автоматически — всё настраивается гибко и под ваши нужды.

## Статус-проекта
! На данный момент готова лишь прототип версия на bash, нужная только для того чтобы пощупать удобство подобного взаимодействия с системой. Сама реализация переносится на `C`

## Как это должно работать

### Файлы в системе

```
~/myproject -> ~/.frost/unpaked/myproject/  | Гибкая ссылка на дирикторию с снапшотом
~/.frost/                                   | 
       ./unpaked/                           | Директория с распоковаными снапшотами
               ./myproject/                 | tmpfs с распокованым снапшотом
               myproject.meta               | Метаданные одноимённого снапшота
       ./snapshots/                         | Директоря для снапшотов
                 ./myproject.tar.gz         | Сохранённый снапшот
```
При перезагрузке размонтируются tmpfs и удаляются ссылки на frost-пространства, как и сами пространства, снапшоты остаются не тронутыми

### Команды

```bash
frost create [name] # Создать новое временное окружение
frost pick [nameї   # Развернуть снапшот (монтируется tmpfs, создаётся ссылка и копируются метаданные)
frost snap [name]   # Сделать снапшот текущего состояния frost-пространства в котором вы находитесь
frost drop [name]   # Удалить окружение (в котором вы находитесь или по имени) (НЕ snapshot)
```
Находясь в развёрнутом окружении можно не вводить name

### Метаданные

Для каждого frost-окружения создаётся файл с метаданными:
```json
{
    "name": "myproject",
    "created": "2026-01-01T12:37:37",
    "modified": "2026-01-01T15:37:37",
    "autopick": true
}
```

`autopick: true` — демон после перезагрузки не будет зачищать остатки директории и сам выполнить frost pick для этого окружения
